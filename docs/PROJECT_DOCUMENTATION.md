# Anjali — Project Documentation

GitHub is the source of truth for this project documentation. Notion indexes this file in the Priyansh App Factory Command Center.

## 00. Executive Summary
Anjali is a respectful devotional iOS app candidate focused on a short daily offering, intention, or prayer gesture. It is for families and individuals who want a calm daily spiritual moment. The end product should be either a standalone app with a clear daily ritual loop or a module inside the broader spiritual product line.

## 01. Product
MVP scope: daily offering screen, intention/theme selection, short guided ritual, completion state, local streak/history, optional reminder. Acceptance criteria: one-minute ritual is clear, content is respectful, and project has a continue/merge/archive decision.

## 02. Design
Warm devotional palette, minimal UI, respectful iconography, family-friendly typography, no flashy gamification. Screens: home, ritual setup, ritual screen, completion, history, settings.

## 03. Frontend Technical
SwiftUI local-first app. Models: Ritual, Intention, PracticeSession, StreakState. Navigation: Today, Ritual, History, Settings. Optional reminders.

## 04. Backend Technical
No backend for v1. Future services may include content library updates, festival content, or optional sync.

## 05. Business
Trust-first spiritual product. Monetization should be gentle: premium content packs, family features, or bundle with a larger spiritual app.

## 06. Marketing
Positioning: a simple daily offering on your iPhone. Channels: community outreach, festival content, family-focused messaging.

## 07. User Acquisition
Beta with families, diaspora users, students, and community contacts. Metrics: first ritual completion, D7 practice, reminder opt-in, repeat ritual rate.

## 08. Execution
Plan: audit repo, run overlap review, decide standalone or merge, freeze ritual MVP, content review, build/QA.

## 09. QA
Test ritual start/completion, streak/history, reminder permission, offline content, reset state, Dynamic Type, VoiceOver, and content review.

## 10. Legal / Compliance
Document local data storage and optional reminders. Review religious content for respectful presentation.

## 11. Operations
Release process: overlap decision, content review, internal build, family/community beta, TestFlight. Post-launch: festival rituals, audio, family mode.
